## 教訓
- **GitHub Pages source branch 很重要** — 要確認 Pages 設定的 source branch 和實際推送的 branch 一致
- **GitHub Contents API 可以處理 ~5MB payload** — 4MB HTML 檔案（base64 編碼後 ~5.3MB）可以成功上傳
- **webFetch 工具會 strip HTML** — 無法用它驗證 base64 data URI，要用 GitHub API 檢查檔案大小來確認
- **deploy-api.js 推送到 main，deploy-git.js 推送到 gh-pages** — 兩個腳本推送的 branch 不同，造成混亂
- **deploy-to-main.js 有 bug** — 它回報 status 200 但實際沒更新，因為 GitHub API 可能回傳 200 但 SHA 沒變。要用 deploy-kinrinko-fix.js 的方式驗證 size 和 SHA
- **base64 嵌入時要驗證每張圖** — fix-remaining.js 會複製附近的圖來填補失敗的下載，導致圖片張冠李戴
